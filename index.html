Rebilly JS SDK Library Reference
The Rebilly JS SDK library allows you to consume the Rebilly API using either Node or the browser.

Pre-Release

The library is not production ready yet. Version 1.0.0 will be the first stable release.

PCI Compliance Note

If you need to handle raw payment card data, you should use the Rebilly Token library to generate tokens for your server-side logic.

Rebilly API Spec

The library is a semantic representation of the Rebilly API spec. A secondary experimental API spec for reports is also available.

 

#Installing

Using npm simply run:

npm install rebilly-js-sdk --save
 

#Usage

To create an instance, you need to provide your secret API key that is available in Rebilly in the Developer > API Keys menu.

Promise Based

Every resource method returns a chainable Promise.

#Configuration

The library authentication can be provided by the apiKey or a session token (JWT). All instantiation parameters are optional.

Example:

const api = RebillyAPI({apiKey: 'secret-api-key', sandbox: true, timeout: 10000});
OptionTypeDescription
apiKeystringYour secret API key. To be used only for server-side integration. See Developer > API Keys.
sandboxbooleanFlag used to enable sandbox mode for the instance. This allows you to run requests without processing real transactions on your account. Defaults to false.
timeoutintegerDefine the timeout in milliseconds for API requests. Defaults to 6000.
versionstringDefine the version of the API to use. Defaults to v2.1. This configuration option does not apply to RebillyExperimentalAPI.
 

#Exposed Modules

The library exposes three modules: the main API, the experimental API and the different errors types.

import RebillyAPI, {RebillyExperimentalAPI, RebillyErrors} from 'rebilly-js-sdk';
ModuleDescription
RebillyAPIMain API wrapper. Used for all common actions except reporting and certain statistics. Based on the Rebilly API spec.
RebillyExperimentalAPISecondary API wrapper. Reserved for experimental endpoints like reports and statistics. This API can break backward-compatibility. Based on the Rebilly Experimental Reports API spec.
RebillyErrorsExposes the different errors returned by both API wrappers.
Importing without Webpack

When importing in an environment without Webpack, the RebillyAPI module wil be exposed as default.

var RebillyAPI = require('rebilly-js-sdk').default;
var RebillyExperimentalAPI = require('rebilly-js-sdk').RebillyExperimentalAPI;
